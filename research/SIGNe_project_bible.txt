📖 THE SIGNe PROJECT BIBLE (v1.0)

Project Overview

    Name: SIGNe

    Environment: Max for Live (Ableton Live), Jitter (OpenGL)

    Core Function: A highly optimized, 3D hardware instancing UI/engine for managing, placing, and controlling Ableton devices/symbols in a 2D/3D viewport.

1. State Management (The Brain)

    All global state is handled by a Max dict named SigneRegistry.

    The dictionary tracks position (x,y), scale, rotation, color, opacity, grouping, and selection status for every spawned instance.

2. Interaction & UI (The Mouse)

    Interaction logic is driven by marquee_selection.js.

    It handles raycasting, marquee drawing, grouping, and humanized quantized movement.

    Routing Rule: The JS script does not talk directly to the Live API. When a user clicks a device, the JS script sends the target ID out its 3rd outlet into a forward object (e.g., send Device_A, selected 1).

3. Ableton Live Integration (The Hooks)

    Architecture: Decentralized.

    The forward object wirelessly broadcasts the selection state.

    Inside each individual device's abstraction/bpatcher, a dynamically named receive object (e.g., r Device_A) catches this broadcast.

    CURRENT BUG: The link between this local receive object and the live.object (which actually triggers set selected_device id in Ableton) is currently broken and failing to focus the device rack.

4. Jitter Instancing Engine (The GPU)

    Uses a jit.gl.mesh driven by a custom GLSL shader (safety_test.jxs / signe_final.jxs).

    Hardware Instancing Slots: To avoid Jitter's internal UV/Normal overriding, we map specific buffers to specific Vertex Attributes:

        VERTEX_ATTR0 = Position (Slot 0)

        VERTEX_ATTR3 = Scale (Slot 3)

        VERTEX_ATTR4 = Color (Slot 4)

        VERTEX_ATTR6 = Symbol Index / Z-Depth (Slot 6)

    Textures: Uses a 3D Texture array (SymbolAtlas) generated by atlas_builder.js to map different visual symbols to different instances.
